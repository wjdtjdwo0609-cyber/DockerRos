=========================================================
DockerRos 웹 시뮬레이터 — Windows 실행 안내
=========================================================

[필요 조건]
1. Python 3.10 이상
   - https://www.python.org/downloads/ 에서 설치
   - 설치 화면에서 "Add Python to PATH" 반드시 체크
2. 최신 Chrome / Edge
3. (PLC 연동 쓸 거면) GX Works2 + GX Simulator2/3 + 정공MES브릿지

[가져갈 폴더]
- DockerRos        ← 이 폴더 (웹 시뮬 + OPC UA 어댑터)
- OPC UA           ← 정공MES브릿지 (PLC ↔ OPC UA, 옵션)
※ DocScope 는 무관하니 가져올 필요 없음

[실행 순서]

(1) 웹 시뮬레이터만 — 단독 실행
    "웹 시뮬.bat" 더블클릭
      → 콘솔 창에 서버 메시지 + 브라우저가 자동으로 열림 (http://localhost:8090)
      → 종료하려면 콘솔 창에서 Ctrl+C 또는 창 닫기

(2) PLC 연동까지 — 풀 시스템
    a. 정공MES브릿지 실행 (Windows: run_bridge.bat 또는 패키징된 exe)
       → OPC UA 서버가 포트 4840에 뜸
    b. "OPC UA 어댑터.bat" 더블클릭
       → 첫 실행 시 venv 생성 + asyncua/websockets 자동 설치 (1-2분)
       → 이후엔 즉시 실행. 포트 9091 에서 WebSocket 대기
    c. "웹 시뮬.bat" 더블클릭
    d. 브라우저 우측 패널에 "OPC UA (PLC 연동) ✓ 연결됨" 뜨면 OK

[GX Works2 테스트 흐름]
1. GX Works2에서 래더 작성 (예: M8000 → Y0030)
2. GX Simulator3 시작
3. 정공MES브릿지가 GX Sim3와 연결됐는지 확인
4. 우리 웹 시뮬에서 시나리오 로드 → Conv1이 Y030 자동 바인딩됨
5. GX Works2 모니터에서 Y0030 ON → 웹 시뮬 컨베이어가 즉시 동작

[자주 묻는 문제]

Q. "Python을 찾을 수 없습니다" 에러
A. PATH 에 Python 추가 안 됨. Python 재설치하면서
   "Add Python to PATH" 반드시 체크.
   설치 후 cmd 에서 "python --version" 으로 확인.

Q. 어댑터가 "OPC UA 연결 끊김" 반복
A. 정공MES브릿지가 안 떠있거나 4840 포트가 막힘.
   브릿지 먼저 실행. Windows 방화벽에서 4840, 8090, 9091 허용.

Q. 캐시 때문에 코드 수정이 반영 안 됨
A. 브라우저에서 Ctrl+Shift+R (하드 리로드).
   서버는 이미 no-cache 헤더 보내고 있어서 일반 새로고침으로도 됨.

Q. 한글 깨짐 (콘솔)
A. .bat 첫 줄에 chcp 65001 (UTF-8) 이미 들어있음.
   그래도 깨지면 Windows Terminal 사용 권장.

[버전 정보]
- web/app.js 버전 쿼리 v=47
- 정공MES브릿지 OPC UA 엔드포인트:
  opc.tcp://127.0.0.1:4840/smartfactory/server/
- 어댑터 WebSocket: ws://127.0.0.1:9091
