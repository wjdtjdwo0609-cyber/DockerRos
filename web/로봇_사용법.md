# 로봇 웹 시뮬레이터 — 로봇 사용법

브라우저 기반 로봇 시뮬레이터(Three.js)에서 **로봇을 조작하는 방법** 정리.
설치·PLC 연동·네이티브 ROS2는 상위 폴더 [README.md](../README.md) 참고.

> ★ 표시 = 디지털 트윈 테스트용으로 새로 추가된 기능 (8·9절).

---

## 0. 빠른 시작

| OS | 실행 방법 |
|---|---|
| macOS | `웹 시뮬.command` 더블클릭 |
| Windows | `웹 시뮬.bat` 더블클릭 |
| 수동 | 터미널에서 `cd web && python3 serve.py 8090` |

브라우저 주소:
- 일반 시뮬레이터 — `http://localhost:8090/`
- 디지털 트윈(임베드, 3D 캔버스만) — `http://localhost:8090/?embed=1`

화면이 안 바뀌면 **Cmd/Ctrl + Shift + R** (캐시 무시 새로고침).

## 1. 화면 구성

```
┌────────────────────────────────────────────────────────────┐
│ 상단 툴바  [+오브젝트…] [🏭 시나리오 로드] [🎲 자동 테스트] … │
├──────────────────────────────────────────┬─────────────────┤
│                                          │ 우측 패널 #panel │
│           3D 뷰포트                       │  로봇 추가/제거   │
│                                          │  카메라 프리셋    │
│   [🏭 시나리오 테스트]  ← 좌측 하단        │  Mode (수동/IK)   │
│      플로팅 버튼 (9절)                     │  툴 / 트레일      │
│                                          │  🧪 디지털트윈테스트│
│                                          │  관절 슬라이더    │
└──────────────────────────────────────────┴─────────────────┘
```

- **우측 패널**에서 로봇을 추가하고 조작한다.
- embed 모드(`?embed=1`)에서는 툴바·우측 패널이 숨겨지고 **좌측 하단 플로팅 버튼만** 보인다.

## 3. 로봇 추가 · 선택 · 제거

1. 우측 패널 → **추가할 로봇 타입** 드롭다운 (Indy7 / Indy12 / UR5e / UR10e / Panda / Fanuc)
2. **[+ 로봇 추가]** 클릭
3. **씬 로봇 목록**에 들어옴 → 목록에서 클릭하면 그 로봇이 **활성 로봇**이 됨
4. 제거: **[− 제거]** (현재 활성 로봇 삭제)

> 모든 조작(슬라이더·IK·테스트 패널)은 **활성 로봇** 하나에만 적용된다.
> 로봇이 여러 대면 먼저 목록에서 대상 로봇을 클릭해 활성으로 만든다.

---

## 4. 로봇 움직이기 — 두 가지 모드

우측 패널 → **Mode** 에서 전환.

### A. 수동 모드 (슬라이더) — 관절 직접 제어
- 하단 **관절** 영역에 관절 수(6~7개)만큼 슬라이더가 생김
- 슬라이더를 드래그하면 해당 관절이 회전 → 3D에 즉시 반영

### B. IK 모드 (드래그) — 끝점 위치만 지정
- 3D 뷰에서 로봇 끝점(그리퍼)을 마우스로 잡고 드래그
- 역기구학이 관절각을 자동 계산해 자연스러운 자세로 이동
- 도달 불가 위치로 끌면 가장 가까운 도달 가능 자세로 수렴

## 5. 자세 프리셋

우측 패널의 버튼 (1.2초 애니메이션으로 부드럽게 이동):

| 버튼 | 자세 |
|---|---|
| 🏠 Home 자세 | 작업 준비 자세 (−π/2 C자형, 그리퍼 아래) |
| ⊙ 영점 자세 | 모든 관절 0° (URDF 영점, 팔 쭉 뻗음) |

## 6. 툴 (End-effector)

우측 패널 → **툴** → 도구 선택:

| 도구 | 설명 |
|---|---|
| 없음 | 맨 플랜지 |
| 그리퍼 | 아래 `opening` 슬라이더로 벌림 폭 조절 (0~0.08 m) |
| 펜 | 끝점이 지나간 자리에 곡선이 그려짐 (드로잉) |

## 7. 경로 트레일

우측 패널 → **경로 트레일** → `📍 트레일 OFF` 토글 →
활성 로봇 끝점 궤적이 색 곡선으로 누적. `🗑️ 지우기`로 초기화.

## 8. ★ 디지털 트윈 테스트 패널

우측 패널 하단 **🧪 디지털 트윈 테스트** 섹션. 활성 로봇의 **베이스 배치**(놓인
위치·방향)를 실시간으로 바꿔보는 테스트용 컨트롤. 시뮬 표시 전용 —
관절·실제 로봇·relay 와 무관하다.

| 컨트롤 | 동작 |
|---|---|
| **베이스 회전 (yaw°)** 슬라이더 | 드래그하면 활성 로봇이 메인축(수직 z축) 기준으로 실시간 회전 (−180°~180°) |
| **위치 X / Y / Z (m)** 입력칸 | 숫자를 바꾸면 로봇이 그 위치로 실시간 이동 |
| **🧪 테스트 회전** 버튼 | 활성 로봇을 메인축 기준 360° 한 바퀴(2.5초) 돌렸다 원위치. 도는 중 다시 누르면 즉시 정지 |
| **↺ 동기화** 버튼 | 활성 로봇의 현재 배치값을 입력칸에 다시 읽어옴 |

**사용 순서**
1. 씬 로봇 목록에서 테스트할 로봇을 클릭 (활성 로봇 지정)
2. yaw 슬라이더를 움직여 방향을, X/Y/Z로 위치를 조정 — 3D에 즉시 반영
3. `🧪 테스트 회전`으로 한 바퀴 스핀 점검
4. 로봇을 바꾸면 패널이 그 로봇 값으로 자동 동기화됨 (수동 동기화는 `↺`)

> "활성 로봇 없음" 메시지가 뜨면 → 3절대로 로봇을 먼저 추가/선택할 것.

## 9. ★ 시나리오 테스트 버튼 (플로팅)

화면 **좌측 하단**에 항상 떠 있는 파란 **`🏭 시나리오 테스트`** 버튼.
embed 모드(디지털 트윈)에서 툴바·패널이 숨겨져도 이 버튼은 보인다.

| 클릭 | 동작 |
|---|---|
| 1번째 | 시나리오(3-로봇 소켓 분류 라인) 로드 → 완료되면 **주문 무한 반복 시작**. 버튼이 빨간 `⏹ 테스트 정지`로 바뀜 |
| 2번째 | 무한 반복 정지, 버튼 원복 |

로딩 중에는 `⏳ 로딩…`으로 비활성화돼 중복 클릭이 막힌다.
한 번 클릭으로 라인 전체가 자동으로 돌아가므로 디지털 트윈 시연에 쓰면 된다.

---

## 10. 시나리오 + 주문 (수동)

플로팅 버튼 대신 단계별로 돌리고 싶을 때:

1. 상단 `🏭 시나리오 로드` — 3-로봇 라인(공급·검사·무게/적재) 자동 배치
2. 상단 `📋 주문 패널` 토글 → 8핀/12핀 개수 입력 → `+ 주문 넣기`
3. Robot1이 매거진에서 픽 → 컨베이어로 공급 → 라인 진행
4. 상단 `🎲 자동 테스트` — 랜덤 주문 무한 투입 (다시 누르면 정지)

## 12. 실제 로봇 연동 (디지털 트윈)

웹 시뮬을 실제 로봇의 **디지털 트윈**으로 쓰는 방법. 실제 로봇의 관절값을
ROS2 토픽으로 받아 3D 로봇이 실시간으로 똑같이 따라 움직인다.

### 12-1. 데이터 흐름

```
 실제 Indy7
    │  IndyDCP3 SDK 로 관절각(rad) 읽기
    ▼
 joint_state_relay            (예: Intel5 loadcell.tools.joint_state_relay)
    │  <cid>_joint_state.json  파일 IPC  또는  직접 ROS publish
    ▼
 ROS2 토픽  /<cid>/joint_states   (sensor_msgs/msg/JointState)
    │
    ▼
 rosbridge   ws://localhost:9090   (ROS2 ↔ WebSocket 다리)
    │
    ▼
 웹 시뮬  구독 → 3D 로봇 관절에 그대로 적용 → 실시간 미러링
```

### 12-2. 토픽 구조

| 토픽 | 대상 |
|---|---|
| `/joint_states` | 단일 로봇 / cid 미태그 씬 (back-compat) |
| `/r1/joint_states` | 라인 로봇 1 (cid = `r1`) |
| `/r2/joint_states` | 라인 로봇 2 (cid = `r2`) |
| `/r3/joint_states` | 라인 로봇 3 (cid = `r3`) |

- 메시지 타입: `sensor_msgs/msg/JointState` — `name[]` + `position[]` 사용.
- **cid 태그**: `🏭 시나리오 로드` 시 R1/R2/R3 에 각각 `r1/r2/r3` cid 가 붙고,
  `/<cid>/joint_states` 가 정확히 그 팔 하나만 구동한다.
- `name[]` 의 관절 이름은 URDF 관절 이름(`joint0`~`joint5`)과 일치해야 적용된다.

### 12-3. 라이브 미러 우선순위

실제 `/<cid>/joint_states` 프레임이 들어오면 그 팔은 최근 **1.5초**간
"라이브 미러" 상태가 되어 **시나리오 자동재생이 일시 양보**한다 (실제 로봇
데이터가 우선). 교시/녹화 데이터 자체는 절대 수정되지 않고 *재생*만 멈춘다.
→ 실제 로봇과 시나리오가 같은 팔을 두고 다투지 않는다.

### 12-4. 연결 단계

```
1) rosbridge 실행 (ROS2 ↔ WebSocket 다리)
   macOS:  ./ROS2\ 브릿지.command  더블클릭
   수동:   scripts/run-rosbridge.sh   → ws://localhost:9090

2) 실제 로봇 관절을 ROS 로 흘리기 — 둘 중 하나
   (a) 실제 Indy7 + 관절 릴레이
       IndyController 가 관절을 읽어 <cid>_joint_state.json 으로 쓰고,
       joint_state_relay 가 그걸 /<cid>/joint_states 로 publish.
   (b) ROS2 노드 / Gazebo 가 /joint_states 를 직접 publish.

3) 웹 시뮬: 우측 패널 → "ROS2 실시간" → [연결]
   → 상태창에 "✓ 연결됨 — /joint_states 구독" 뜨면 OK
   → 이제 3D 로봇이 실제 로봇을 그대로 따라 움직임

4) embed 모드(?embed=1, 디지털 트윈 박스)는 시나리오 로드 후
   [연결]을 자동으로 눌러줌 — 수동 클릭 불필요
```

### 12-5. 양방향 동기화

| 방향 | 동작 | 용도 |
|---|---|---|
| 실제 → 시뮬 | `/<cid>/joint_states` 구독 → 3D 로봇이 따라감 | 디지털 트윈 (실로봇 마스터) |
| 시뮬 → 실제 | 수동 모드에서 슬라이더로 관절을 움직이면 `/joint_states` 로 publish | 시뮬을 마스터로 쓸 때 |

### 12-6. 컨베이어 · 센서 · PLC 연동

관절이 아니라 컨베이어 토글 / 센서 / 비전 I/O 를 PLC(OPC UA·MC Protocol)와
엮는 것은 별도 경로다 → [../README.md](../README.md) 의 "🚀 사용법" **2절
(웹 시뮬 + PLC 연동)** 참고. 태그 매핑표·단계별 실행이 거기 정리돼 있다.

### 12-7. 연동 트러블슈팅

| 증상 | 원인 / 해결 |
|---|---|
| `❌ rosbridge 실행 중인지 확인` | rosbridge 안 떠 있음 → 12-4의 1번 먼저 실행 |
| 연결은 됐는데 안 움직임 | 실제 로봇/노드가 토픽을 실제로 publish 하는지 확인: `ros2 topic echo /r1/joint_states` |
| 특정 팔만 안 따라옴 | cid 불일치 — 시나리오 로봇 cid(`r1/r2/r3`)와 토픽 이름이 같은지 확인 |
| 3D 로봇이 떨리거나 튐 | `/joint_states`(shared)와 `/<cid>/joint_states`가 같은 팔에 동시 적용 — cid 태그를 붙이면 per-robot 토픽만 따름 |
| 일부 관절만 적용됨 | `msg.name[]`이 URDF 관절 이름(`joint0`~`joint5`)과 달라서 — 이름을 맞출 것 |

## 13. 자주 막히는 것

| 증상 | 해결 |
|---|---|
| 버튼/패널이 안 보임 | embed 모드(`?embed=1`)면 의도된 동작 — 좌측 하단 플로팅 버튼만 보임. 전체 UI는 `?embed=1` 빼고 접속 |
| 변경한 게 화면에 없음 | `Cmd/Ctrl + Shift + R` 하드 리로드 |
| "활성 로봇 없음" | 우측 패널 → 씬 로봇 목록에서 로봇을 먼저 클릭 |

| 테스트 회전이 안 멈춤 | `🧪 테스트 회전` 버튼을 한 번 더 클릭 (토글) |
